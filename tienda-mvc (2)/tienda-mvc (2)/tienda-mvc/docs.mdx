# Tienda MVC

Sistema CRUD de productos desarrollado en PHP puro siguiendo el patrón **MVC** (Model-View-Controller), con autenticación de administrador, catálogo público con búsqueda, footer responsivo de Bootstrap y **API REST** de productos en formato JSON.

## Base de datos

Crea la base de datos y las tablas ejecutando el siguiente SQL:

```sql
CREATE DATABASE IF NOT EXISTS tienda_mvc
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE tienda_mvc;

CREATE TABLE usuarios (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    username        VARCHAR(50)  NOT NULL UNIQUE,
    password        VARCHAR(255) NOT NULL,
    nombre_completo VARCHAR(100) NOT NULL
);

CREATE TABLE productos (
    id            INT AUTO_INCREMENT PRIMARY KEY,
    sku           VARCHAR(50)   NOT NULL UNIQUE,
    nombre        VARCHAR(150)  NOT NULL,
    descripcion   TEXT          NOT NULL,
    precio_compra DECIMAL(10,2) NOT NULL,
    precio_venta  DECIMAL(10,2) NOT NULL,
    existencia    INT           NOT NULL DEFAULT 0
);
```

### Crear usuario administrador

Ejecuta `hash.php` en tu servidor para obtener el hash bcrypt de `admin123`:

```bash
php hash.php
```

Copia el hash generado e insértalo:

```sql
INSERT INTO usuarios (username, password, nombre_completo)
VALUES ('admin', '$2y$12$HASH_GENERADO_AQUI', 'Administrador');
```

<Warning>
  Elimina `hash.php` del servidor después de crear el usuario. Nunca lo dejes en producción.
</Warning>

## Rutas disponibles

El enrutador es `index.php` (front controller). Todas las rutas se pasan por el parámetro GET `?route=`.

### Rutas web

| Ruta | Método | Descripción | Requiere sesión |
|------|--------|-------------|-----------------|
| `catalogo` | GET | Catálogo público de productos | No |
| `login` | GET | Formulario de inicio de sesión | No |
| `auth/login` | POST | Procesa el login | No |
| `logout` | GET | Cierra la sesión | Sí |
| `productos` | GET | Lista de productos (admin) | Sí |
| `productos/create` | GET | Formulario de nuevo producto | Sí |
| `productos/store` | POST | Guarda nuevo producto | Sí |
| `productos/edit&id=N` | GET | Formulario de edición | Sí |
| `productos/update` | POST | Actualiza producto existente | Sí |
| `productos/delete` | POST | Elimina producto | Sí |

### Rutas API REST

| Ruta | Método | Descripción | Auth |
|------|--------|-------------|------|
| `api/productos` | GET | Lista completa de productos en JSON | No |
| `api/productos/show&id=N` | GET | Un producto por ID en JSON | No |
| `api/productos/buscar&q=termino` | GET | Búsqueda por nombre o descripción | No |

## API REST

La API devuelve datos en formato **JSON** con cabeceras CORS habilitadas para consumo desde cualquier origen.

### Estructura de respuesta exitosa

```json
{
  "status": "success",
  "total": 3,
  "data": [
    {
      "id": 1,
      "sku": "ABC-001",
      "nombre": "Producto ejemplo",
      "descripcion": "Descripción del producto",
      "precio_compra": "10.50",
      "precio_venta": "19.99",
      "existencia": 100
    }
  ]
}
```

### Estructura de respuesta de error

```json
{
  "status": "error",
  "mensaje": "Producto no encontrado."
}
```

### Endpoint 1 — Listado completo

```
GET index.php?route=api/productos
```

Devuelve todos los productos registrados en la base de datos.

**Ejemplo de consumo con JavaScript (fetch):**

```javascript
fetch('http://localhost/tienda-mvc/index.php?route=api/productos')
  .then(res => res.json())
  .then(data => {
    console.log('Total de productos:', data.total);
    data.data.forEach(p => console.log(p.nombre, p.precio_venta));
  });
```

### Endpoint 2 — Producto por ID

```
GET index.php?route=api/productos/show&id=1
```

| Parámetro | Tipo | Requerido | Descripción |
|-----------|------|-----------|-------------|
| `id` | int | Sí | ID del producto a consultar |

**Códigos HTTP posibles:**

| Código | Significado |
|--------|------------|
| `200` | OK — producto encontrado |
| `400` | ID inválido (≤ 0) |
| `404` | Producto no encontrado |

### Endpoint 3 — Búsqueda

```
GET index.php?route=api/productos/buscar&q=zapato
```

| Parámetro | Tipo | Requerido | Descripción |
|-----------|------|-----------|-------------|
| `q` | string | No | Término de búsqueda (nombre o descripción) |

Si `q` está vacío, devuelve todos los productos.

## Módulos principales

### Config\\Database

Gestiona la conexión PDO con MySQL.

```php
$db  = new \Config\Database();
$pdo = $db->connect(); // Devuelve instancia PDO
```

**Opciones configuradas automáticamente:**
- `ERRMODE_EXCEPTION` — lanza excepciones en errores SQL
- `FETCH_ASSOC` — resultados como arrays asociativos
- Charset `utf8mb4`

---

### Config\\Autoload

Registra un autoloader con `spl_autoload_register` que convierte namespaces a rutas de archivo:

```
Controllers\ProductoController  →  controllers/ProductoController.php
Controllers\ApiController       →  controllers/ApiController.php
Models\ProductoModel            →  models/ProductoModel.php
Config\Database                 →  config/Database.php
```

<Note>
  El primer segmento del namespace se convierte a minúsculas para que coincida con el nombre de la carpeta (`Controllers` → `controllers`).
</Note>

---

### Models\\ProductoModel

| Método | Parámetros | Retorna | Descripción |
|--------|-----------|---------|-------------|
| `obtenerTodos()` | — | `array` | Todos los productos ordenados por ID DESC |
| `buscarPublico(string $termino)` | `$termino` | `array` | Búsqueda LIKE por nombre o descripción |
| `obtenerPorId(int $id)` | `$id` | `array\|null` | Un producto por ID |
| `crear(array $data)` | `$data` | `bool` | Inserta producto usando transacción PDO |
| `actualizar(int $id, array $data)` | `$id`, `$data` | `bool` | Actualiza producto usando transacción PDO |
| `eliminar(int $id)` | `$id` | `bool` | Elimina producto usando transacción PDO |

**Campos esperados en `$data`:**

```php
$data = [
    'sku'           => 'ABC-001',
    'nombre'        => 'Producto ejemplo',
    'descripcion'   => 'Descripción del producto',
    'precio_compra' => 10.50,
    'precio_venta'  => 19.99,
    'existencia'    => 100,
];
```

Todos los métodos de escritura usan **transacciones PDO** con `beginTransaction()`, `commit()` y `rollBack()` para garantizar integridad de datos.

---

### Models\\UsuarioModel

| Método | Parámetros | Retorna | Descripción |
|--------|-----------|---------|-------------|
| `buscarPorUsername(string $username)` | `$username` | `array\|null` | Usuario por nombre de usuario |

---

### Controllers\\AuthController

| Método | Descripción |
|--------|-------------|
| `showLogin()` | Renderiza `views/auth/login.php` |
| `login()` | Valida credenciales con `password_verify()`, inicia sesión |
| `logout()` | Destruye la sesión y redirige al login |

La sesión de administrador se almacena en `$_SESSION['admin']` con las claves `id`, `username` y `nombre_completo`.

---

### Controllers\\ProductoController

Todos los métodos llaman internamente a `verificarSesion()`. Si no hay sesión activa, redirigen a `login`.

| Método | Descripción |
|--------|-------------|
| `index()` | Lista todos los productos |
| `create()` | Muestra formulario de creación |
| `store()` | Valida y guarda producto nuevo |
| `edit()` | Carga producto por ID y muestra formulario |
| `update()` | Valida y actualiza producto existente |
| `delete()` | Elimina producto por ID (POST) |

**Validaciones en `store()` y `update()`:**
- Todos los campos son obligatorios
- `precio_compra`, `precio_venta` y `existencia` deben ser numéricos
- No se permiten valores negativos

---

### Controllers\\ApiController

Controlador de la **API REST**. No requiere sesión (acceso público de lectura).

| Método | Ruta | Descripción |
|--------|------|-------------|
| `productos()` | `api/productos` | Devuelve todos los productos en JSON |
| `productoShow()` | `api/productos/show&id=N` | Devuelve un producto por ID |
| `productosBuscar()` | `api/productos/buscar&q=texto` | Búsqueda por nombre o descripción |

**Cabeceras enviadas en toda respuesta de la API:**

```
Content-Type: application/json; charset=utf-8
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, OPTIONS
```

---

### Controllers\\PublicController

| Método | Descripción |
|--------|-------------|
| `catalogo()` | Obtiene productos (filtrados si viene `?buscar=`) y renderiza el catálogo |

## Correcciones aplicadas al código original

### 1. `edit.php` — Atributo `action` mal escrito

```html
<!-- ❌ Original -->
<form acton="index.php?route=productos/update" method="POST">

<!-- ✅ Corregido -->
<form action="index.php?route=productos/update" method="POST">
```

### 2. `edit.php` — Espacio en blanco en `<textarea>`

```html
<!-- ❌ Original (genera espacio inicial en el valor guardado) -->
<textarea ...> <?= htmlspecialchars($producto['descripcion']); ?></textarea>

<!-- ✅ Corregido -->
<textarea ...><?= htmlspecialchars($producto['descripcion']); ?></textarea>
```

### 3. `catalogo.php` — Clase CSS con carácter inválido

```html
<!-- ❌ Original -->
<input type="text" name="buscar" cñass="form-control" ...>

<!-- ✅ Corregido -->
<input type="text" name="buscar" class="form-control" ...>
```

### 4. `catalogo.php` — Clase de Bootstrap incorrecta

```html
<!-- ❌ Original -->
<h5 class="card-tittle">...</h5>

<!-- ✅ Corregido -->
<h5 class="card-title">...</h5>
```

### 5. `ProductoController.php` — Espacio extra en cabecera `Location`

```php
// ❌ Original (el espacio rompe la redirección HTTP)
header('Location : index.php?route=login');

// ✅ Corregido
header('Location: index.php?route=login');
```

### 6. `ProductoController.php` — URL con `=?` invertido

```php
// ❌ Original
header('Location: index.php=?route=productos/create');

// ✅ Corregido
header('Location: index.php?route=productos/create');
```

### 7. `ProductoController.php` — Typos en mensajes de sesión

```php
// ❌ Original
$_SESSION['success'] = 'Producto actualizaco correctamente.';
$_SESSION['success'] = 'Producto elimindo correctamente.';
$_SESSION['error']   = 'No fue posble eliminar el producto.';
$_SESSION['error']   = 'No se permiten valres negativos.';

// ✅ Corregido
$_SESSION['success'] = 'Producto actualizado correctamente.';
$_SESSION['success'] = 'Producto eliminado correctamente.';
$_SESSION['error']   = 'No fue posible eliminar el producto.';
$_SESSION['error']   = 'No se permiten valores negativos.';
```

### 8. `ProductoModel.php` — `bindParam` sin `:` en `actualizar()`

```php
// ❌ Original (el parámetro no tiene el prefijo : → PDO no lo enlaza)
$stmt->bindParam('existencia', $data['existencia'], PDO::PARAM_INT);

// ✅ Corregido
$stmt->bindParam(':existencia', $data['existencia'], PDO::PARAM_INT);
```

### 9. `header.php` — Navbar sin clase `navbar`

```html
<!-- ❌ Original (Bootstrap no reconoce el componente) -->
<nav class="navbar-expand-lg navbar-dark bg-dark">

<!-- ✅ Corregido + toggler responsivo añadido -->
<nav class="navbar navbar-expand-lg navbar-dark bg-dark">
```

### 10. `footer.php` — Faltaba Bootstrap JS Bundle

```html
<!-- ✅ Añadido antes de </body> (necesario para toggler, alertas, etc.) -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.8/dist/js/bootstrap.bundle.min.js"
  integrity="sha384-YvpcrYf0tY3lHB60NNkmXc4s9bIOgUxi8T/jzmQ0BVM8YCTkYMRomH2dByG+3D+q"
  crossorigin="anonymous"></script>
```

## Seguridad

<Warning>
  Este proyecto es educativo. Para un entorno de producción se recomienda:
</Warning>

- Usar variables de entorno (`.env`) para las credenciales de la base de datos
- Añadir tokens **CSRF** en todos los formularios POST
- Implementar **HTTPS**
- Llamar a `session_regenerate_id(true)` tras el login exitoso para prevenir session fixation
- Agregar autenticación por **token (Bearer / API key)** a los endpoints de la API REST
- Eliminar `hash.php` del servidor tras crear el usuario administrador
- Restringir `Access-Control-Allow-Origin` a dominios específicos en producción
